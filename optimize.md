# Оптимизация проекта BeanStore

## 1. Для чего нужна оптимизация

Оптимизация необходима для:

- Увеличения скорости загрузки сайта — влияет на удержание пользователей и SEO.
- Улучшения пользовательского опыта (UX) — фильтры (цена, наличие, категория, обжарка) должны работать мгновенно.
- Снижения нагрузки на браузер — особенно на мобильных устройствах.
- Повышения конверсии — быстрый сайт продаёт лучше.
- Чистоты и поддерживаемости кода — чтобы легко добавлять новые товары или фильтры.

## 2. Что будем оптимизировать

В проекте BeanStore (9 карточек товара, 4 группы фильтров) оптимизируем:

| Компонент | Что именно |
|-----------|-------------|
| HTML | семантика, удаление пустых атрибутов src у изображений |
| CSS | сжатие, удаление неиспользуемых стилей, оптимизация теней и шрифтов |
| JavaScript | фильтрацию товаров (по цене, наличию, категории, обжарке) |
| Изображения | добавление реальных изображений, форматы, размеры, ленивая загрузка |
| Рендеринг | уменьшение перерисовок при смене фильтров |

## 3. Как будем оптимизировать

### HTML

Проблемы в текущем коде:
- У 6 из 9 карточек отсутствует атрибут src у изображения (`<img src="">`).
- Нет атрибута `loading="lazy"` для изображений.
- Фильтры используют чекбоксы, но у них нет привязки к данным товаров.

Что исправляем:
- Добавить реальные пути к изображениям или временные заглушки.
- Добавить `loading="lazy"` для всех изображений.
- Добавить дата-атрибуты к карточкам товаров для фильтрации (например, `data-category`, `data-roast`, `data-price`, `data-stock`).

### CSS

Проблемы в текущем коде:
- Стили вынесены в отдельный файл `styles.css` (хорошо), но требуется минификация.
- Возможно, есть повторяющиеся правила для карточек.

Что исправляем:
- Минификация CSS (удалить пробелы, комментарии, лишние переносы).
- Объединить повторяющиеся стили карточек.
- Критические стили (шапка, фильтры) оставить в style, остальное подгружать позже.

### JavaScript

Проблема в текущем коде:
- В файле `script.js` скорее всего нет реализованной фильтрации или она реализована неоптимально.
- При каждом изменении фильтра заново перебираются все 9 карточек.

Как оптимизируем:

1. Кэширование DOM — сохранить массив карточек при загрузке.
2. Фильтрация без пересоздания — не удалять и не создавать карточки заново, а просто скрывать/показывать через display: none.
3. Делегирование событий — один обработчик на все фильтры.
4. Оптимизация ползунка цены — использовать debounce при движении ползунка.

Пример быстрой фильтрации:

// Плохо (медленно)
function filter() {
  productsContainer.innerHTML = '';
  filteredProducts.forEach(p => productsContainer.appendChild(p.card));
}

// Хорошо (быстро)
function filter() {
  products.forEach(card => {
    const matches = checkFilters(card);
    card.style.display = matches ? 'block' : 'none';
  });
}

### Изображения

Проблемы в текущем коде:
- У 6 карточек пустой src.
- Изображения могут быть в тяжёлых форматах.

Что исправляем:
- Добавить реальные изображения или хотя бы плейсхолдеры.
- Конвертировать фото кофе в WebP.
- Прописать явные width и height.
- Включить loading="lazy".

### Загрузка ресурсов
- JS скрипт — уже в конце body (хорошо).
- Шрифты — подключены через Google Fonts (хорошо), но добавить font-display: swap.
